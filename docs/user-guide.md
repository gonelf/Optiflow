## Getting Started

### Creating Your Account

1. Visit [optivibe.app/signup](https://optivibe.app/signup)
2. Enter your email address and choose a secure password
3. Verify your email address
4. Complete your profile setup

### Setting Up Your First Workspace

After signing up, you'll be prompted to create your first workspace:

1. **Workspace Name**: Choose a name that represents your project or company
2. **Workspace Slug**: This will be part of your workspace URL (e.g., `my-company`)
3. **Plan**: Select your plan (Free, Pro, or Enterprise)

### Inviting Team Members

To collaborate with your team:

1. Go to **Settings** ‚Üí **Team**
2. Click **Invite Member**
3. Enter their email address
4. Select their role:
   - **Owner**: Full control over workspace
   - **Admin**: Can manage pages and settings
   - **Member**: Can create and edit pages
   - **Viewer**: Read-only access

## Creating Your First Page

### Option 1: Start from Scratch

1. Click **Create Page** on your dashboard
2. Enter a **Title** (e.g., "Product Launch Landing Page")
3. Set a **Slug** (e.g., "product-launch") - this will be your page URL
4. Click **Create**

### Option 2: Use a Template

1. Click **Create Page**
2. Select the **Templates** tab
3. Browse available templates by category:
   - Landing Pages
   - Pricing Pages
   - Coming Soon Pages
   - Demo Request Pages
4. Click **Use Template** on your preferred option
5. Customize the title and slug
6. Click **Create**

### Option 3: Generate with AI

1. Click **Create Page**
2. Select the **AI Generate** tab
3. Describe your page in detail:
   ```
   Example: "Create a landing page for a B2B SaaS project management tool.
   Target audience: Project managers and team leads.
   Include a hero section, features section, pricing table, and testimonials."
   ```
4. Click **Generate**
5. Wait 5-10 seconds for AI to create your page
6. Review and customize the generated page

## Using the Page Builder

### Interface Overview

The page builder consists of four main areas:

1. **Top Toolbar**: Save, preview, and publish options
2. **Left Sidebar**: Component palette
3. **Center Canvas**: Visual page editor
4. **Right Panel**: Component properties and settings

### Adding Components

1. Click **Add Component** or drag from the sidebar
2. Choose from 8+ component types:
   - **Hero**: Eye-catching header with headline and CTA
   - **Features**: Showcase product features in a grid
   - **Pricing**: Display pricing tiers and plans
   - **Testimonials**: Customer reviews and social proof
   - **CTA**: Call-to-action buttons and sections
   - **FAQ**: Frequently asked questions accordion
   - **Form**: Lead capture and contact forms
   - **Newsletter**: Email signup widget

### Editing Components

1. Click on any component to select it
2. Use the right panel to edit:
   - **Content**: Text, images, links
   - **Styles**: Colors, spacing, typography
   - **Settings**: Component-specific options

### Reordering Components

- **Drag and drop**: Click and drag components to reorder
- **Move buttons**: Use up/down arrows in the component menu

### Duplicating Components

1. Select the component
2. Click the **Duplicate** button
3. The copy will appear below the original

### Deleting Components

1. Select the component
2. Click the **Delete** button (trash icon)
3. Confirm the deletion

## Setting Up A/B Tests

### Creating Your First A/B Test

1. Open your published page
2. Click **Create A/B Test**
3. Name your test (e.g., "Headline Test - Q1 2024")
4. Set your **Primary Goal**:
   - Form submission
   - Button click
   - Page scroll depth
   - Custom event
5. Configure **Traffic Split**:
   - Control: 50%
   - Variant A: 50%
   - (Or custom percentages)
6. Click **Create Test**

### Creating Variants

1. Your original page becomes the **Control**
2. Click **Create Variant**
3. Make changes to test:
   - Different headline
   - Alternative CTA text
   - Different hero image
   - Layout variations
4. Save your variant

### Running Your Test

1. Review your test setup
2. Click **Start Test**
3. Traffic will automatically be split between variants
4. Monitor results in the **A/B Tests** dashboard

### Interpreting Results

OptiVibe shows you:

- **Conversion Rate**: Percentage of visitors who completed the goal
- **Confidence Level**: Statistical significance (aim for 95%+)
- **Sample Size**: Number of visitors per variant
- **Winner Recommendation**: Based on statistical analysis

### When to Declare a Winner

Wait until:
- ‚úÖ Confidence level reaches 95% or higher
- ‚úÖ Minimum sample size is met (typically 1,000+ visitors)
- ‚úÖ Test has run for at least one week

## Understanding Analytics

### Real-Time Dashboard

View current activity:
- **Active Visitors**: People on your page right now
- **Today's Stats**: Page views, conversions, bounce rate
- **Top Pages**: Most visited pages
- **Traffic Sources**: Where visitors are coming from

### Conversion Funnels

Track the user journey:

1. Go to **Analytics** ‚Üí **Funnels**
2. Click **Create Funnel**
3. Define steps:
   - Step 1: Page view
   - Step 2: Button click
   - Step 3: Form submission
4. Analyze drop-off rates at each step

### Heatmaps

Visualize user behavior:

1. Go to **Analytics** ‚Üí **Heatmaps**
2. Select your page
3. Choose heatmap type:
   - **Click Heatmap**: Where users click
   - **Scroll Heatmap**: How far users scroll
4. Use insights to optimize placement

### Cohort Analysis

Compare different user groups:

1. Go to **Analytics** ‚Üí **Cohorts**
2. Group users by:
   - Signup date
   - Traffic source
   - Geographic location
3. Compare conversion rates across cohorts

## Publishing Your Pages

### Saving as Draft

- Click **Save** to save changes without publishing
- Drafts are private and only visible to workspace members

### Previewing Your Page

1. Click **Preview** to see how it looks
2. Test on different devices:
   - Desktop
   - Tablet
   - Mobile
3. Make adjustments as needed

### Publishing

1. Click **Publish** when ready
2. Review your page settings:
   - SEO metadata
   - Custom domain (optional)
   - Analytics tracking
3. Click **Confirm**
4. Your page is now live!

### Accessing Your Published Page

Your page will be available at:
- `https://optivibe.app/p/your-slug` (default)
- `https://your-domain.com` (with custom domain)

### Unpublishing

To take a page offline:
1. Open the published page
2. Click **Unpublish**
3. Confirm

## Advanced Features

### Custom Domains

Connect your own domain:

1. Go to **Settings** ‚Üí **Domains**
2. Click **Add Domain**
3. Enter your domain (e.g., `pages.mycompany.com`)
4. Add the provided DNS records to your domain provider
5. Wait for verification (usually 24-48 hours)

### Webhooks

Integrate with external services:

1. Go to **Settings** ‚Üí **Integrations** ‚Üí **Webhooks**
2. Click **Create Webhook**
3. Enter webhook URL
4. Select events to trigger:
   - Page published
   - Conversion created
   - A/B test completed
   - Form submitted
5. Save and test

### Stripe Integration

Accept payments on your pages:

1. Go to **Integrations** ‚Üí **Stripe**
2. Click **Connect Stripe**
3. Authorize OptiVibe
4. Add payment buttons to your pages
5. Track revenue in analytics

### API Access

Programmatic access:

1. Go to **Settings** ‚Üí **API Keys**
2. Click **Create API Key**
3. Give it a name and set permissions
4. Use the API key in your applications
5. See [API Documentation](./api-documentation.md) for details

## Best Practices

### Page Design

‚úÖ **Do:**
- Keep headlines clear and benefit-focused
- Use high-quality images
- Include clear calls-to-action
- Make forms short and simple
- Ensure mobile responsiveness
- Test across browsers

‚ùå **Don't:**
- Overload with too many components
- Use generic stock photos
- Hide important info "below the fold"
- Make text too small
- Use too many different fonts/colors

### A/B Testing

‚úÖ **Do:**
- Test one variable at a time
- Run tests for sufficient time (7-14 days)
- Wait for statistical significance
- Document your learnings
- Test high-impact elements first

‚ùå **Don't:**
- End tests too early
- Test multiple variables simultaneously
- Ignore statistical significance
- Run too many tests at once
- Make assumptions without data

### SEO Optimization

‚úÖ **Do:**
- Write unique, descriptive titles (50-60 characters)
- Craft compelling meta descriptions (150-160 characters)
- Use descriptive image alt text
- Include relevant keywords naturally
- Ensure fast page load times
- Make pages mobile-friendly

### Analytics

‚úÖ **Do:**
- Set clear conversion goals
- Review analytics weekly
- Track UTM parameters in campaigns
- Create custom funnels for key journeys
- Use heatmaps to optimize layout

## Troubleshooting

### Common Issues

#### Page Won't Publish

**Problem**: "Publish" button is disabled

**Solutions**:
- Ensure page has a title and slug
- Check that page has at least one component
- Verify workspace is not over plan limits

#### Images Not Loading

**Problem**: Images appear broken

**Solutions**:
- Check image URL is publicly accessible
- Use HTTPS URLs only
- Ensure images are under 5MB
- Verify image format (JPG, PNG, WEBP, SVG)

#### A/B Test Not Starting

**Problem**: Test status stuck on "Draft"

**Solutions**:
- Ensure page is published
- Create at least two variants
- Set a primary goal
- Check traffic split adds up to 100%

#### Analytics Not Tracking

**Problem**: Zero pageviews showing

**Solutions**:
- Verify page is published (not draft)
- Check browser doesn't block tracking
- Wait 5-10 minutes for data to appear
- Ensure page is receiving traffic
